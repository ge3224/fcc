Basics

Getting Started

HTML tags give a webpage its structure. You can use HTML tags to add photos, buttons, and other elements to your webpage.

HTML elements have opening tags like <h1> and closing tags like </h1>.

The text for an element goes between its opening and closing tags.

Here’s an example:

        <h1>My Title</h1>

Headings

The h1 through h6 heading elements are used to signify the importance of content below them. The lower the number, the higher the importance, so h2 elements have less importance than h1 elements. Only use one h1 element per page and place lower importance headings below higher importance headings. 

        
          <h1>Page Title</h1>

          <h2>The Next Level of Importance</h2>

          <h3>...Third</h3>

          <h4>...Fourth</h4>

          <h5>...Fifth</h5>

          <h6>...Sixth</h6>

Paragraphs

The p element is used to create a paragraph of text on websites.

        <p>This is a paragraph</p>

Paragraphs

Commenting allows you to leave messages without affecting the browser display. It also allows you to make code inactive. A comment in HTML starts with <!--, contains any number of lines of text, and ends with -->.

For example, examine the following comment, which contains the text “TODO: Remove h1”:

        
          <!-- TODO: Remove h1 -->

The following example shows how the elements we’ve discussed so far work together.

        
          <h1>Title</h1>

          <p>An introductory paragraph of text</p>

          <h2>Section Title</h2>

          <p>A new paragraph of text</p>

          <!-- Continue learning -->

In review:

  1. <tag>Text</tag> - opening and closing tags
  2. h1, h2, h3, h4, h5, and h6 - heading tags
  3. Lower heading numbers signify greater importance.
  4. p - a paragraph tag
  5. <!-- Comment -->
Organizing Content

HTML5

HTML5 has some elements that identify different content areas. These elements make your HTML easier to read and help with Search Engine Optimization (SEO) and accessibility.

The main Element

One of the new content elements introduced in HTML5 is main. Identify the main section of a page by adding a <main> opening tag before the first element of the page’s content and a </main> closing tag after the final content element. 

        
          <main>

            <!-- content elements -->

            <h1>Title</h1>

            <p>Paragraph</p>

          </main>

In the snippet above, the content is inside the main element. This is called nesting. Nested elements should be placed two spaces further to the right of the element they are nested in. This spacing is called indentation and it is used to make HTML easier to read.

The section Element

Separate segments of content by surrounding each in a section element.

When you add a lower rank heading element to the page, it’s implied that you’re starting a new subsection.

Let’s take a look at some other common content elements: images and hyperlinks.

Images

You can add images to your website by using the img element. img elements have an opening tag without a closing tag. A tag for an element without a closing tag is known as a self-closing tag.

        
          <img>

          <!-- Or -->

          <img />

HTML attributes are special words used inside the opening tag of an element to control the element’s behavior. The src attribute in an img element specifies the image’s URL (where the image is located).

An example of an img element using an src attribute:

        <img src="https://www.example.com/the-image.jpg">.

All img elements should have an alt attribute. The alt attribute’s text is used for screen readers to improve accessibility and is displayed if the image fails to load.

For example, the following example has an alt attribute with the text “A cat”.

        <img src="cat.jpg" alt="A cat">

The figure Element

Another new element introduced in HTML5 is the figure element. The figure element represents self-contained content and will allow you to associate an image with a caption.

A figcaption element is used to add a caption to describe the image contained within the figure element. For example, <figcaption>A cute cat</figcaption> adds the caption A cute cat.

Links

You can link to another page with the anchor a element. For example, the next snippet would link to freecodecamp.org.

        <a href="https://freecodecamp.org"></a>

A link’s text must be placed between the opening and closing tags of an anchor a element. For example, <a href="https://www.freecodecamp.org">click here to go to freeCodeCamp.org</a> is a link with the text click here to go to freeCodeCamp.org.

You can surround words inside in another element with anchor tags and turn them into a link. You can also turn an image into a link by surrounding it with anchor tags.

Add a target attribute with the value “_blank” to the a element’s opening tag, so that the link opens in a new tab.

More Common Tags

Text Emphasis (Italicize)

Emphasize a word in an element, such as figcaption, by wrapping it in an emphasis em element.

Text Importance (Embold)

The strong element is used to indicate that some text is of strong importance or urgent.

        
          <p>This is a paragraph of text with a <em>emphasized word</em> and an
          <strong>emboldened
          word</strong></p>

Lists

Unordered lists, such as bulleted lists, are surrounded by ul elements.

Each individual list item is surrouned by li elements to create items in a list. Here is an example of list items in an unordered list:

        
          <ul>

            <li>milk</li>

            <li>cheese</li>

          </ul>

The code for an ordered list (ol) is similar to an unordered list, but list items in an ordered list are numbered when displayed.

        
          <ol>

            <li>bread</li>

            <li>jam</li>

          </ol>

HTML Document

All pages should begin with the following document declaration: <!DOCTYPE html>. This special string is known as a declaration and ensures the browser tries to meet industry-wide specifications.

Immediately below the document declaration, each page should start with an opening <html> tag.

The last line of the document should have a closing </html> tag.

Within the opening <html> tag, the lang attribute should be included to specify the language of the page. For example, a value of “en” could be used to indicate English.

All page content elements that should be rendered to the page go inside the page’s body element. 

        
          <!DOCTYPE html>

          <html lang="en">

            <body>

              <!--Content goes here-->

            </body>

          </html>

Meta Data

All content elements that should be rendered to the page go inside the body element. However, other important information goes inside the head element.

The title element determines what browsers show in the title bar or tab for the page.

You can set browser behavior by adding self-closing meta elements in the head. Here’s an example:

        
          <!DOCTYPE html>

          <html lang="en">

            <head>

              <title>Page Title Here</title>

              <meta charset="utf-8">

              <meta name="viewport" content="width=device-width, initial-scale=1.0">

            </head>

            <body>

              <!--Content to be displayed-->

            </body>

          </html>

Forms

Data is submitted by means of a form element. The action attribute indicates where form data should be sent. The following snippet tells the browser that the form data should be sent to the path /submit-url. 

        <form action="/submit-url"></form>

Inputs

The input element allows you several ways to collect data from a web form. Like img elements, input elements are self-closing and do not need closing tags.

Attribute: type

There are many kinds of inputs you can create using the type attribute. You can easily create a password field, reset button, or a control to let users select a file from their computer.

        
          <form action="/submit-url">

            <input type="text" />

            <input type="radio" />

          </form>

You can use radio buttons for questions where you want only one answer out of multiple options.

Attribute: name

In order for a form’s data to be accessed by the location specified in the action attribute, you must give the text field a name attribute and assign it a value to represent the data being submitted.

For example, you could use the following syntax for an email address text field:

        <input type="text" name="email">

Multiple radio buttons can be selected at the same time. To make it so selecting one radio button automatically deselects the other, all buttons in a group must have a name attribute with the same value.

Attribute: value

If you select a radio button and submit the form, the form data for the button is based on its name and value attributes. Add a value attribute to define data that should be submitted for each radio button. 

        
          <input type="radio" name="choice" value="chocolate" />

          <input type="radio" name="choice" value="vanilla" />
Attribute: placeholder

Placeholder text is used to give people a hint about what kind of information to enter into an input.

        <input type="text" placeholder="Email address">
Attribute required

To prevent a user from submitting your form when required information is missing, you need to add the required attribute to an input element. There’s no need to set a value to the required attribute. Instead, just add the word required to the input element, making sure there is space between it and other attributes.

        <input type="text" placeholder="Email address" required />
Attribute: id

The id attribute is used to identify specific HTML elements. Each id attribute’s value must be unique from all other id values for the entire page.

More On Forms

Checkboxes

Forms commonly use checkboxes for questions that may have more than one answer. For example, here’s a checkbox with the option of tacos:

        <input type="checkbox" />

While you won’t notice this in the browser, adding a name attribute to each checkbox input element makes it easier for a server to process your web form, especially when there are several checkboxes.

Like radio buttons, form data for selected checkboxes are name / value attribute pairs. While the value attribute is optional, it’s best practice to include it with any checkboxes or radio buttons on the page.

Labels

label elements are used to help associate the text for an input element with the input element itself (especially for assistive technologies like screen readers). For example, <label><input type="radio"> cat</label> makes it so clicking the word cat also selects the corresponding radio button.

Attribute: for

There’s another way to associate an input element’s text with the element itself. You can nest the text within a label element and add a for attribute with the same value as the input element’s id attribute.

        
          <form>

            <label for="e_1">Example 1: <input type="checkbox" id="e_1" name="examples" value="1"
          /></label>

            <label for="e_2">Example 2: <input type="checkbox" id="e_2" name="examples" value="2"
          /></label>

          </form>
Attribute: checked

In order to make a checkbox checked or radio button selected by default, you need to add the checked attribute to it. There’s no need to set a value to the checked attribute. Instead, just add the word checked to the input element, making sure there is space between it and other attributes.

        
          <form>

            <label for="c1">First Choice: <input type="checkbox" id="c1" name="choices" value="1"
          checked /></label>

            <label for="c2">First Choice: <input type="checkbox" id="c2" name="choices" value="2"
          /></label>

            <label for="c3">First Choice: <input type="checkbox" id="c3" name="choices" value="3"
          /></label>

          </form>

Buttons

Use the button element to create a clickable button. For example, the following code creates a button with the text “Click Here”.

        <button>Click Here</button>

Both input and button elements are inline elements, which don’t appear on new lines.

A button in a form will submit the form by default. However, relying on default behavior may cause confusion, so it’s best practice to add the type attribute with the value “submit” to the button to make it clear that it is a submit button.

        
          <form>

            <label for="o1"><input type="radio" id="o1" name="options" value="1" /> Option
          1</label><br>

            <label for="o2"><input type="radio" id="o2" name="options" value="2" /> Option
          2</label><br>

            <button type="submit" value="Submit">Submit</button>

          </form>
